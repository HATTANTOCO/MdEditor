# MdEditor for baserCMS 4

baserCMS 4系（CakePHP 2系ベース）の管理画面に、高機能な Markdown エディタ **EasyMDE** を導入するプラグインです。  
標準の CKEditor と置き換える、または出し分ける形で、ブログ記事や固定ページを快適な Markdown 記法で執筆できるようになります。

## 主な特徴

- **安心のデータ保存・プレビュー設計（PHPタグ自動保護）**
  固定ページの保存時やプレビュー時に発生しやすい、baserCMSコア側とのシステム衝突を完全に回避します。
  文章の中にPHPコードの解説（`<?php` などの記述）が含まれていたり、閉じ忘れなどの不完全な構文があっても、システムが裏側で自動的に安全なコードへと一時変換して保護。サイト内検索のインデックス更新時など、バックグラウンドの処理を巻き込んで画面が真っ白（システム停止）になるトラブルを未然に防ぎます。

- **管理画面でのスムーズな再編集 ＆ 本文エリアだけの正確なMarkdown変換**
  管理画面を開いた際には、保護されていたコードが自動的に元のクリーンな記述へと復元されるため、ストレスなくそのまま再編集が可能です。
  一方、公開画面の表示時には、記事全体のHTMLから「Markdownで書かれた本文詳細エリア」だけをシステムがピンポイントで正確に切り抜いてHTMLへと変換。コメント欄やフッターなどの既存のテーマ部品のデザインを崩すことなく、安全かつ独立してMarkdown表示を組み込むことができます。

- **ドラッグ＆ドロップによる画像非同期アップロード**
  EasyMDE の標準機能と完全に連動し、エディタへの画像ドラッグ＆ドロップまたはペーストによって、プラグイン固有のカスタムエンドポイント（`MdEditorUploadsController`）を介した非同期画像アップロード及びエディタへのパス自動挿入が可能です。

- **テーマ自動同化型の多軸シンタックスハイライト**
  フロントエンド表示用に `Highlight.js`（atom-one-lightテーマ）および専用のコアスクリプト・CSSを自動注入。プログラミングのコードブロック（` ``` `）は、縦スクロール時はコードと行番号が完全に1対1で連動し、横スクロール時だけ行番号が左端に固定される「X軸限定 sticky 吸着構造」を採用。さらに `color-mix` を駆使したカメレオン設計により、将来どのような配色テーマ（Monokai等）を追記しても、文字や枠線が背景の不透明度を保ったまま全自動で美しく同化します。

## インストール要件

* CMS本体: baserCMS v4.7.x以上 admin-second/ admin-third対応
* 動作環境: PHP 8.x以上

## インストール方法

1. 本リポジトリをダウンロードし、フォルダ名を `MdEditor` に変更します。
2. baserCMS の `app/Plugin/` ディレクトリ配下に配置します。
3. baserCMS 管理画面の「プラグイン管理」より、**MdEditor** を有効化します。

## 使い方・設定

### 標準エディタからの切り替え
baserCMS 4系標準の CKEditor と競合を避けるため、システム設定、または各固定ページ・ブログの設定等で「エディタタイプ」を調整するか、特定の入力欄のみに適用されるよう出し分けを行ってください。

### 画像挿入について
本プラグインは、動作の安定性を最優先し、コアファイルやファイルアップローダー（Uploaderプラグイン）への無理なシステムハックを排除しています。「本文はMarkdownを実直に書き、画像ボタンからはEasyMDE標準のURL入力プロパティを呼び出す」という、コンフリクト（フリーズや404）を起こさない安全なフォールバック設計をあえて選択しました。
よって、baserCMS標準のファイルアップローダー（Uploaderプラグイン）とは連携せず（というか連携させることができませんでした）、独立した仕様としています。
これは、**baserCMS4系コアシステムおよびUploaderプラグインと標準搭載のCKEditorとの強力な依存、制限**を安全に回避し、機能を完全に独立させ、快適性、安定性を優先したためです。
尚、ファイルアップローダー（Uploaderプラグイン）自体は、管理画面の **「アップロード管理」** から問題なく利用できますので、アップロードしたファイルのURLを使用し、EasyMDE標準の「画像ボタン」から呼び出したURL入力プロパティにペーストするなどしてください。

そのかわり、EasyMDE の標準機能と連動して、エディタへの画像ドラッグ＆ドロップまたはペーストによる非同期画像アップロードに対応しています。

### カスタマイズ設定
ツールバーの表示項目などは、`Config/setting.php` より変更が可能です。
デフォルトでは以下の主要なツールのほか、カスタムアイコンの「infoボックス」が定義されています：
`["bold", "italic", "strikethrough", "heading", "|", "quote", "code", "table", "horizontal-rule", "|", "unordered-list", "ordered-list", "|", "link", "image", "|", "preview", "side-by-side", "fullscreen", "|", "guide"]`

### ブログ記事一覧やサイト内検索結果での記述露出について（重要）
ブログの一覧ページ（インデックス/アーカイブ）や、サイト内検索結果ページ、各種ウィジェットでは、テーマの設計やコアの仕様によって「概要（content）」だけでなく、EasyMDEで記述した「本文詳細（detail）」から一律に文字数を切り出して出力するケースがあります。
そのため、対策を行わない場合、一覧や検索結果の画面に以下の記述がそのまま剥き出しで露出してしまう原因になります。

- **生の Markdown 記法**（`###` や ` ``` `、`:::info` などの特殊構文）
- **本プラグイン専用のアンカーマーカー**（`<!--MDE_BODY_START-->` や `<!--MDE_BODY_END-->`）
- **無害化保護用の独自コード**（固定ページや検索インデックス生成時のクラッシュを防ぐために、プラグインが一時的に変換している `<?php` などのシールドトークン）

**【推奨される対策・運用ルール】**
ブログ記事を執筆する際は、本文詳細とは別にある **「概要（説明文）」欄への要約テキストの入力をマスト（必須）** として運用してください。
概要欄を活用することで、以下の絶大なメリットが得られます：

1. **表示の完全なクリーン化**：Markdown記号や保護用コード、隠しマーカーが一切混ざらない、クリーンな日本語の抜粋文で一覧・検索結果画面を100%統一できます。
2. **プラグインの軽量化維持**：システム側で無理に一覧画面や検索結果のバッファまで横取りしてParsedownパースを掛ける必要がなくなり、エディタプラグイン本来の「超軽量・高安定」な設計を損なわずに運用できます。
3. **SEO・SNSシェアの最適化**：概要欄のテキストは、検索エンジンのメタディスクリプションや SNS シェア時の OGP 紹介文として自動的に再利用されるため、サイトの評価向上に直結します。

## コアロジック（技術的アプローチ）

本プラグインは、baserCMS 4系における自動クレンジングや検索インデックス作成時の割り込み処理による影響を完全に回避し、かつテーマ全体のレイアウト崩壊を100%防ぐため、以下のタイミングでデータの調停とフィルタリングを行っています。

1. **データ送信・保存時（`Controller.initialize`）**
   管理画面での保存・プレビュー（`POST` / `PUT`）を最上流で検知した段階で、文章内の実行用 PHP タグ（`<?php` や `?>`）を対象に、バックトラック制限による文字漏れリスクが一切ない高速な文字列置換（`str_replace`）を用いて `<?php` などの独自コードへと一時的に無害化（シールド）。
   さらにブログ記事（`detail`）の保存時には、前後に他の要素と絶対に地続きにならない完全な独立行としてアンカーマーカー（`<!--MDE_BODY_START-->`）をデータベースへ直接焼き付けます。

2. **管理画面のエディタフォーム描画時（`Controller.beforeRender`）**
   コアの初期化処理がすべて完了した描画直前（`beforeRender`）のタイミングをピンポイントでフック。無害化されていた独自コードを元のクリーンな PHP タグへと正確に復元してからビューに渡すことで、管理画面（EasyMDE）上でのシームレスな再編集を可能にします。

3. **フロント公開画面の最終出力時（`Helper.beforeLayout`）**
   画面全体の組み立てが行われ、テーマの全レイアウト（ヘッダー、フッター、コメント欄など）が合体した直前のバッファ（`Blocks->get('content')`）をフック。
   * **固定ページ（Pages）**: 独自コードの安全性を維持したまま、改行コードをクリーンアップして `CustomParsedown` へ引き渡し、ページ全体を安全にHTML化します。
   * **ブログ詳細ページ（archives）**: 脆弱な正規表現（`preg_match`）を完全に排除し、PHP標準の高速な文字列検索（`strpos` / `substr`）を採用。データベースに焼き付けたアンカーマーカーの間（＝純粋な本文詳細エリア）だけをミリ単位の誤差もなく正確に切り抜き（スライス）。前後に明示的な空行をドッキングして `CustomParsedown` に放り込むことで、近接するコメント欄やフッターなどのテーマ部品へのパース影響を物理的に100%分離・遮断し、ファイル名言語ヘッダー付きの美しい HTML へと出力します。
